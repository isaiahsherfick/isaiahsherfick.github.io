1. Setup

To get started, you're not actually going to do too much coding yet. What this initial module will cover is just setting up your development environment (most of which I've already done for you) and familiarizing yourself with some of the tools you'll be using. Now, the curriculum I have in mind for you is a little unconventional. Most coding courses start out really holding your hand and letting you stay in cozy little Windows land where everything has a graphical interface and you can use a mouse.... We're not gonna do that here. We're going back to the 70's to start out. Trust me that this will all be incredibly helpful in the long run though it may seem weird and unfamiliar at first.

The gist

You're going to be using the secure shell protocol to remotely connect to a small computer running in my house. You'll do this via the git bash terminal emulator, which you can download here. Then you're going to play around and familiarize yourself with the environment before we get into too much coding. We're doing it this way because the alternative is to either have you install linux on your computer or have you do all this stuff in Windows, which would require you to install and download like 50 things which is just an annoying amount of setup. I've got all of that set up on this little server already and I can put stuff there for you to mess with very easily.

Step 1: download and install git bash

Download it here. Don't worry about what "git" is yet. It's important, but we're not there yet. For now, this is just a terminal (little typey hacker screen) you can use to remotely connect to the dev environment.

Step 2: use git bash to ssh into the server

  1. Text me to get the IP address of the server as well as your login credentials.
  2. Open up git bash and type in the following command (I'm using placeholder stuff for login / ip here): " ssh jack@111.222.333.444 "
  3. If all goes well and you've successfully entered your password, you've now successfully connected to the remote server. Welcome to Linux land!

Step 3: create a new directory (folder) and move into it

  1. Type the following command: " mkdir jack_working " and press enter.
  2. Now type "ls" and press enter. This will show you all of the files and folders in your current directory. This is just the current folder that you're in. It's exactly the same as how Windows folders work -- there's just no mouse or anything. You should see the directory you created in the previous step.
  3. Now type "cd ja" and press tab (this will autocomplete for you) and hit enter.
  4. Verify that you're in your new directory by executing the command " pwd " (print working directory). If you see "/home/jack/jack_working", you've done it all right.

Step 4 (last step): create your first C file and run it!

Some people might call me insane for teaching you C as your first introduction to programming, but I stand by it. C is one of the oldest and most widely used programming languages, but it is also BARE BONES . This is the language that ATMs and stoplights and coffee makers use mostly these days, but it's sort of where it all began. It's really picky, it's really """"hard"""", and it can be quite annoying at times. However, it's going to be a big payoff when you get to use a modern language and you'll be able to truly appreciate all of the heavy lifting they silently do for you.

  1. Type the following command: " touch hello.c " and press enter. Then run "ls" again to verify that you did, in fact, create a file called hello.c.
  2. Now type in the following command: " nano hello.c " and run it. You have now opened up your empty hello.c file in the nano text editor. This is the only time I'll have you use nano because it kind of sucks, but I don't want to throw you too far off the deep end right away since this is a really quick task. Think of this exactly like notepad in Windows land - use the arrow keys to move around and type in whatever you want to type. Control+O saves the file, Control+X closes the file when you're done. Don't worry about learning any other shortcuts because the next module will be focused on the best text editor, Vim.
  3. Type the following at the top of your file: "#include <stdio.h>" then open up a new line by hitting enter. You've just imported a library, which is just a bunch of code that somebody else already wrote that's very useful to you. Don't think too hard about it, just consider this a necessary step for now. Do this automatically when you start writing a C file from this point on.

  4. Now we're going to define the main() function. I'm throwing out a bunch of shit right now that we will go into more detail on later, but just roll with it for now. Type in the following code.

    int main(void)

    {

    }

    Inside of these curly braces is what code will be run when we actually execute this code. It's empty right now, so if we were to compile (crunch the code down into a ".exe" type file that you can run), NOTHING would happen because there is nothing inside of the main function. So let's make it do something.
  5. Add the following code into your main function so that it looks like this:

    #include<stdio.h>

    int main(void)

    {

     printf("Hello, world!\n");

    }

  6. Save and exit the file by pressing ctrl+O, ctrl+X.
  7. Now you're back in terminal land. Type the following to compile your code:

    " gcc h " hit tab so it autocompletes, then hit enter. Run "ls" again and you should see a new file: "a.out". This is your compiled code.

  8. FINAL STEP: execute your code!

    run the following command: "./a.out"

    The terminal should say "Hello world!" back to you. You've successfully written a C program and executed it on a remote Linux server. Text me when you're done.

Linux Command References:

  • ls -> show all files in the current directory
  • pwd -> "print working directory"; show where you currently are in the filesystem.
  • cd [/path/to/target/directory] -> "change directory"; change where you are to where you want to go
  • cat [file] -> dump out the contents of [file] to the console
  • vim [file] -> open a file in vim (more soon)
  • nano [file] -> open a file in nano
  • mkdir [new directory name] -> create a new directory in the current working directory
  • rm [file] -> DELETE a file (super permanent, no recycling bin, that shit's gone)
